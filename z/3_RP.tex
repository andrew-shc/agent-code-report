\section{Terminology \& Algorithm Reference}

All code and documentation now use \textbf{ReSIR} (spatial-only, no
temporal reuse). C++ class names retain ``ReSTIR'' for codebase
compatibility (\texttt{PathReSTIR}).

\subsection{Algorithm~3: WRS as RIS with Streaming}

Algorithm~3 in Bitterli et al.\ 2020 describes weighted reservoir sampling
(WRS) as a streaming implementation of RIS. The key insight: RIS generates $M$
candidates $\{x_i\}$ from a source density $p$, weights each by $w_i =
\hat{p}(x_i)/p(x_i)$, then picks $y=x_k$ with probability $w_k / \sum w_i$.
The estimator is unbiased with weight $W_y = \bigl(\sum w_i\bigr) /
\bigl(M \cdot \hat{p}(y)\bigr)$. WRS makes this streaming: candidates are
processed one at a time with accept/reject against a running sum $\texttt{ws}$.
This is exactly what our C++ implementation does at
\texttt{path\_restir.cpp:213--245} --- see \S\ref{sec:cpp}.

\subsection{Algorithm~6: Unbiased Balance-Heuristic Combination}

ReSIR's spatial reuse (Algorithm~4) combines $K$ neighbor reservoirs. The
standard ``cheap'' combination weight is $w_k = W_k \cdot f_q(y_k)$ where $W_k$
is the neighbor's RIS weight. This is biased when the neighbor's sample
visibility differs at the current pixel. Algorithm~6 provides an unbiased
generalization: $w_k = M_k \cdot f_q(y_k)$, using the raw candidate count $M_k$
instead of the weight sum. Our implementation toggles between these via the
\texttt{m\_unbiased} flag (\texttt{path\_restir.h:32}):

\begin{lstlisting}[language=C++,basicstyle=\footnotesize\ttfamily]
FloatC w_nbr = select(ank && nbr_f > Epsilon,
                      (m_unbiased ? nbr_M : nbr_ws) * fq_s / nbr_f, FloatC(0.f));
\end{lstlisting}

\noindent When \texttt{unbiased=true}, the balance-heuristic weight $M_k \cdot
f_q(y_k) / \hat{p}_k(y_k)$ is used, matching Algorithm~6.

\subsection{evalC Explained}

The neighbor's sample is re-evaluated at the current pixel's shading
point via \verb|bsdf_array->evalC(...)| (\texttt{path\_restir.cpp:301})---a
single shadow ray + BSDF eval, not a full re-render. The \texttt{C} suffix
is Dr.Jit convention: \texttt{C}=constant/no-AD, \texttt{D}=AD-tracked.

\section{Code Modularity}

\texttt{restir\_core.py} (281~lines) consolidates shared scaffolding---
\verb|ReSIRConfig|, PSDRJITConnector patches, \verb|PathReSIRFunction|
(3 backward modes), \verb|PathReSIRRenderer|, and mesh helpers---
eliminating~200+ duplicated lines. \verb|backward\_mode| controls the
gradient integrator: \texttt{'full'} (ReSIR with spatial resampling,
1~spp), \texttt{'path'} (PathTracer ablation), \texttt{'none'} (debug).
The neighbor loop tracks \verb|drawn\_nbrs| to prevent duplicate pixel
selection; the final weight counts only visible neighbors via
\verb|select(ank, nbr\_M, FloatC(0.f))|. A clean config-driven experiment
runner (\texttt{cornell_definitive.py}, 298~lines) replaces the previous
812-line benchmark.

for (int c = 0; c < m_n_candidates; ++c) {
    PositionSampleC ps = scene.sample_emitter_position<false>(...);

    FloatC w     = select(ps.pdf > Epsilon, f_hat / ps.pdf, FloatC(0.f));

    MaskC accept = (next_1d() * res_ws) < w && active_di;

\paragraph{Findings.}
ReSIR matches Standard PT in L1 loss (within $0.00009$) using 16$\times$
fewer backward rays (1~vs.\ 16~spp), at~1.7$\times$ lower throughput
(0.12~vs.\ 0.21~it/s). The effective ray budget is 1~spp with
neighbor sharing, giving an effective SPP of $1 + (n_{\text{neighbors}}
\times\text{accept\_rate}) \approx 1.1$---the spatial reuse path carries
negligible independent information on this all-diffuse scene.

\textbf{LR sensitivity.} At $\text{lr}=3$, Adam under-drives ReSIR's
noisier gradients, producing a~$+3.8$~unit Y overshoot at 50~iters.
At $\text{lr}=10$, momentum overshoots the optimum ($-6.9$~units),
having passed through~$Y=83.0$ around iteration~30. The correct
convergence---$Y=83.0$ at $\text{lr}=10$, $30$~iters---matches
Standard PT ($Y=83.1$). A sweep over $n_{\text{neighbors}} \in
\{1,2,4,8\}$ (not shown) produced identical L1 and Y ($\pm1.5$),
confirming spatial resampling is pure overhead on diffuse-only
content: ReSIR reduces to~1~spp PathTracer with appropriately tuned LR.

\textbf{Implication.} On diffuse scenes, ReSIR's $2\times$ speedup
vs.\ standard PBIR (see \S\ref{sec:time}) comes from~1~spp backward
samples, not spatial resampling. The specular benefit---where spatial
reuse reduces gradient variance for glossy-to-specular paths---remains
to be measured on Stanford-ORB.
